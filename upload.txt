① 장바구니에 담기 (add)
수정한 파일을 "이번에 올릴 거야!"라고 지정하는 단계입니다.

Bash
# 특정 파일 하나만 담을 때
git add upload.txt

git add .

② 결제 도장 찍기 (commit)
내 컴퓨터 저장소에 "이 시점의 기록을 저장해!"라고 확정하는 단계입니다.

Bash
git commit -m "첫 번째 파일 업로드 테스트"

주의: 여기서 nothing to commit이 뜨면 파일이 저장되지 않았거나 수정된 내용이 없다는 뜻입니다. 꼭 파일을 저장(Ctrl+S) 하고 시도하세요!

③ 택배 보내기 (push)
내 컴퓨터에 저장된 기록을 GitHub 서버로 실제로 쏘아 올리는 단계입니다.

Bash
git push origin main
(만약 예전 방식의 저장소라면 main 대신 master를 써야 할 수도 있습니다.)

nothing to commit: 모든 작업이 완료되어 서버로 보낼 준비가 됨.

2. 해결 방법: 주소를 다시 정확하게 설정하기
먼저 연결된 주소를 오리님의 저장소 주소로 초기화해야 합니다. 아래 명령어를 순서대로 입력해 보세요.

Bash
# 1. 기존에 잘못 연결된 'origin' 연결을 끊습니다.
git remote remove origin

# 2. 오리님의 정확한 저장소 주소로 다시 연결합니다.
git remote add origin https://github.com/engstar/itest.git

# 3. 다시 파일을 올립니다.
git push origin main
